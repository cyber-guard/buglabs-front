FAQ

If you didn't find an answer here, get in touch via the information below.

So how will this actually help me?

This platform allows you to consistently monitor the changes in infrastructure of your chosen bug bounty programs, so that you can quickly identify and conduct manual testing on those newly discovered targets. It allows you to focus on the important part; the manual part of testing, whilst making sure you don't miss out on any new assets.

What types of tests are available?

At the moment there are 3 test that you can periodically run: sub-domain evaluation, DNS monitoring, and ping sweeps. Thus you can hourly check for new sub-domains, DNS changes, and the ping status of servers/ips.

How frequently can I run each test?

The shortest frequency is one hour, or subsequently 2, 6, and 12 hours. Presumably I will add more frequent options, the limit of which is mainly dependent on the available infrastructure...

Is this a finished product?

Definitely not. This version is essentially pre-alpha stage, and the reason for publishing it in current state is in order to get feedback and suggestions for improvements.

Is the code open-source?

Not yet, for several reasons; firstly the whole code base is written in Perl 5 (Mojolicious framework in case somebody cared), thus I am somewhat sceptical it would catch on with other developers. However more importantly, the platform currently has quite a lot of external dependencies, which means that it is difficult to package and not very efficient. I might change my mind as the platform evolves.

How can I send in suggestions/comments/feedback?

You can use the contact form below or email directly to info[at]buglabs.me

Pricing
